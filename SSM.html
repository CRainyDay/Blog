3.配置项目的web.xml

①关于整合

  我们有两种整合方法:

  • ①同时配置Spring IOC和Spring MVC IOC

      通常情况下, 我们都是配置两个IOC容器的.

    • 将类似于数据源、事务、 整合其他框架的配置都是放在 Spring 的配置文件中, 还有 Service 和 Dao 对象等.
    • 而 Spring MVC 的 IOC 只负责HandlerMapping 映射策略、各种 Controller 控制器、各种视图解析、拦截器、LocalResolver 国际化处理等的配置.

  • ②只配置Spring MVC IOC: 将所有配置都放在 SpringMVC 的配置文件中. 也可以分多个 Spring 的配置文件, 然后使用 import 节点导入其他的配置文件.

  • 在这里, 我们用第①种方法.

②在web.xml中配置字符过滤器和RESTFUL风格API过滤器

  • 为了解决在项目运行过程中出现的乱码问题, 需要配置字符过滤器, 将所有的编码格式统一为 UTF-8.

  • 为了支持 RESTFUL API开发, 需要配置 RESTFUL 风格 API 过滤器.
